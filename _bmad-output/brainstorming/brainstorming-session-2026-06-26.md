#### Schicht 2: Die Drei Modi

**Modus 1 – Fast Refine („Pimp my Prompt"):**
- Roher Prompt rein → lokales LLM → klarer, grammatikalisch sauber, strukturiert
- Keine Ziel-LLM-Adaption, keine Domäne, keine Rückfragen
- <500ms Latenz – das erste Erfolgserlebnis

**Modus 2 – Fast Mode („Full Power, One Shot"):**
- User beschreibt WAS er will → NLP erkennt Domäne → Ziel-LLM-Adapter → optimierter Prompt
- Ein Schuss, keine Rückfragen
- Domänen-Auto-Erkennung macht Schubladen unsichtbar

**Modus 3 – Master Mode („Dialog-Schmiede"):**
- Tool führt Mini-Dialog (2-5 Rückfragen) zur Intentions-Schärfung
- Jede Antwort verfeinert den Prompt
- Erst bei User-Zufriedenheit → Ziel-Adapter → Output
- Tool wird zum Prompt-Engineering-Coach

#### Schicht 3: Dynamische Prompt-Transformation

**Pipeline:** Roher Prompt → Domain Tags → Intent Parse → LLM Adapter → Formatter

**LLM-spezifische Adapter:**
- Claude 3.5: XML-Tags + Chain-of-Thought (`<task>...</task><context>...</context>`)
- GPT-4o: Markdown-Hierarchie (`# Role\n## Context\n## Task`)
- Gemini 1.5: Few-Shot-JSON-Style (`Task: ...\nExamples:\n- ...\nNow: ...`)
- Llama 3: Instruction-Tags (`[INST] System: ...\nUser: ... [/INST]`)
- Mistral: Direkt, minimalistisch, kein Overhead

#### Schicht 4: Skill Tree (Gamification)

**3-Äste-Fähigkeitenbaum (1 Ast = 1 Modus):**

Jeder Ast hat 4 Tiers:
- Tier 1: Entdecker (2-5 Nutzungen) → Modus freischalten
- Tier 2: Kenner (8-10) → Feature freischalten (Warum-Box, Auto-Erkennung, Rückfragen)
- Tier 3: Experte (15-20) → Erweitertes Feature (Tonfall-Slider, LLM-Vergleich, Chain-of-Thought)
- Tier 4: Großmeister (30-50) → Creator-Tools (Eigene Regeln, Custom Maps, Adapter bauen)

**Quest-Typen:**
- Tutorial-Quests: „Verbessere deinen ersten Prompt"
- Tages-Quests: „Nutze heute 3 verschiedene Ziel-LLMs"
- Meilenstein-Quests: „10 perfekte Prompts in Folge"
- Experiment-Quests: „Vergleiche denselben Prompt mit Claude vs. GPT"
- Bau-Quests: „Erstelle deinen eigenen Domänen-Template"

#### Schicht 5: Prompt-Sezierer (Didaktik)

**Farbcodierte Prompt-Dekonstruktion:**
- 🟦 ROLLE: Gibt dem LLM eine Perspektive
- 🟩 AUFGABE: Klare Aufgabe = klares Ergebnis
- 🟨 FORMAT: Output-Format steuern
- 🟧 KONTEXT: Bestimmt Tiefe & Tonfall

**Vorher/Nachher-Diff-Ansicht:**
- Zeigt was hinzugefügt, geändert, entfernt wurde
- Visuell, nicht textuell

#### Schicht 6: First-Run-Flow

**Zero-to-First-Prompt in <60 Sekunden:**
- Tool öffnen → lokales LLM lädt im Hintergrund („Deine KI wird vorbereitet")
- Demo-Prompt sichtbar während Download
- Kein Account, keine Konfiguration nötig

### Key Breakthroughs (Morphological Analysis)

- **Tauri > Electron:** Für lokale LLM-Inferenz ist Tauris Rust-Backend Electron überlegen (Performance, Binary-Größe, Sicherheit)
- **Hybrid ist Pflicht:** Ohne lokales LLM scheitert Non-Tech-Onboarding; ohne Cloud-API sind Master Mode und Ziel-Adapter nicht sinnvoll machbar
- **3-Modi-Architektur-Kopplung:** Die 3 Modi beziehen ihre Daseinsberechtigung aus der Hybrid-LLM-Architektur – jeder Modus korrespondiert mit einer Routing-Entscheidung
- **Skill-Tree = Architektur-Spiegel:** Der 3-Äste-Baum ist keine aufgeklebte Gamification, sondern ein strukturelles Abbild der 3-Modi-Architektur
- **Prompt-Sezierer als didaktischer Kern:** Nicht Textwände, sondern farbcodierte, klickbare Prompt-Dekonstruktion lehrt „Warum"

- **S2 – Austauschbares lokales Modell:** Llama 3.2 1B (oder andere) statt fest verdrahtetem Phi-3 Mini. Modell-Austauschbarkeit als User-Feature designen.
- **S4 – Python-Sidecar für NLP:** Python-Sidecar-Prozess (spaCy, transformers) für Domänen-Erkennung statt Rust-Eigenbau. Hybrid: Rust (Performance-Kern) + Python (NLP-Komfort).
- **S5 – Community-Challenges + Top-Down-Quests:** Hybrid-Gamification – vorgegebene Quests UND User-erstellte Challenges.

**Rejected:** S3 (CSS-only statt Framer Motion)

---

### Lens 2: COMBINE

**Accepted:**

- **C1 – Lern-Sezierer (Sezierer + Skill-Tree):** Der Prompt-Sezierer wird kontext-sensitiv zum Skill-Level. Tier-1 sieht 2 Farben, Tier-4 sieht 7+ Ebenen mit CoT-Sichtbarkeit. Didaktik und Gamification verschmolzen.
- **C2 – Lern-Boost (Fast Mode + Vorher/Nachher-Diff):** Jeder Fast-Mode-Durchlauf zeigt automatisch den Diff. Didaktik als passiver Nebeneffekt der Kernfunktion.
- **C4 – Prompt-Marktplatz (Plugin-System + Community-Templates):** User teilen komplette „Prompt-Blueprints" (Domäne + Ziel-LLM + Template). „Steam Workshop für Prompts."

**Rejected:** C3 (Adaptiver Router nach Skill-Level), C5 (Domänen-Coach via NLP)

---

### Lens 3: ADAPT (aus anderen Domänen)

**Accepted:**

- **A1 – Prompt-Inspiration-Feed (von Spotify Discover Weekly):** Tool schlägt wöchentlich Prompt-Ideen vor basierend auf genutzten Domänen. Proaktive Inspiration.
- **A2 – Streak-Freeze (von Duolingo):** 1× pro Woche Streak-Erhaltungs-Joker, verdient durch Quests. Motiviert ohne zu bestrafen.
- **A3 – Prompt-Kommando-Palette (von VSCode):** `Ctrl+Shift+P` = universelle Befehlssuche. Unsichtbare Power-User-Schicht.
- **A4 – Prompt-Blueprint-Bibliothek (von Figma Community Files):** Figma-artiges Browsing-Erlebnis für Prompt-Strukturen. Vorschau + „Remix"-Button.

**Rejected:** A5 (Notion-Slash-Commands)

---

### Lens 4: MODIFY

**Accepted:**

- **M3 – Ziel-LLM-Adapter visuell:** Statt unsichtbarer Transformation zeigt eine „Adapter-Vorschau" live, wie der Prompt für Claude vs. GPT vs. Gemini aussieht. Adaption wird zum Vergleichs- und Lern-Feature.

**Prüfpunkte (technische Validierung nötig):**

- **M1 – Master-First:** Wizard-artiger Master-Dialog als allererste Erfahrung. Fast Refine/Fast Mode werden als „Abkürzungen" freigeschaltet. Stärkster Modus zuerst = stärkster erster Eindruck.
- **M4 – WASM-Llama:** Lokales LLM via WebAssembly direkt im WebView (kein Download). Zero-Install innerhalb von Tauri. Weniger Performance, null Wartezeit.

**Rejected:** M2 (Stufenloser Slider statt Modi), M5 (Sezierer als Standalone-App)

### Lens 5: PUT TO OTHER USES

**Accepted:** Keine. Alle Ideen (P1–P5) verworfen – Fokus auf Kernprodukt.

**Rejected:** P1 (Schul-Plug-in), P2 (Gamification-SDK), P3 (Adapter-Wissensdatenbank), P4 (Recruiting-Simulator), P5 (Intent-Engine-API)

---

### Lens 6: ELIMINATE

**Accepted:**

- **E1 – Video-Tutorials streichen:** Frage-Storming-Ergebnis: User lesen/schauen nicht. Inline-Erklärungen + Sezierer + Learning-by-Doing ersetzen alle Video-Tutorials.
- **E2 – Badges+Level streichen:** Skill-Tree bildet Fortschritt bereits ab. Separate Badges sind redundant und fühlen sich „kindisch" an. Ein Gamification-System statt zwei.
- **E3 – Manuelle Domänen-Auswahl streichen:** Nur Auto-Erkennung + Community-Templates. Kein Dropdown. User wird nie gefragt „In welcher Domäne denkst du?"
- **E4 – Fast Mode als separaten Modus streichen:** 2 Modi statt 3: Fast Refine (Polieren, lokal) + Master Mode (Tief, dialogisch, Cloud). Fast-Mode-Funktionen (Adaption, Domänen) integriert in beide. Weniger kognitive Last.
- **E5 – LLM-Wahl für Tier-1+2 streichen:** Kein „Wähle dein LLM"-Dropdown beim Start. Universeller Adapter für Einsteiger. LLM-Vergleich als Tier-3-Power-Feature.

**Rejected:** Keine.

---

### Lens 7: REVERSE

**Accepted:**

- **R1 – User trainiert das Tool (lernendes System):** Tool lernt aus User-Entscheidungen. User überschreibt Auto-Erkennung → Tool passt Gewichtung an. User ändert Adapter-Output → Tool merkt sich Muster. Persönlicher werdendes Tool.
- **R2 – LLM schreibt eigenen Adapter (Meta-Prompting):** Meta-Prompt ans Ziel-LLM: „Beschreibe, wie ich dich optimal nach X frage." Antwort wird zum Prompt-Template. Keine menschliche Annahme.

**Rejected:** R3 (Schwer→Einfach-Default), R4 (Prompts suchen User), R5 (Sabotage-Modus)

2. **Lernendes System** (R1) – Tool adaptiert sich an User-Präferenzen
3. **LLM-selbstgeschriebene Adapter** (R2) – Meta-Prompting statt statischer Templates
4. **Lern-Sezierer** (C1) – Didaktik passt sich Skill-Level an
5. **Prompt-Marktplatz** (C4) + Blueprint-Bibliothek (A4) – Community-getriebene Content-Plattform
6. **Python-Sidecar** (S4) – NLP-Komfort per spaCy, nicht Rust-Eigenbau
7. **Adapter-Vorschau live** (M3) – Transparenz und Lern-Werkzeug statt unsichtbarer Magie
8. **Gamification-Verschlankung** (E2, E5) – Nur Skill-Tree + Quests/Challenges, keine Badges/Level/LLM-Dropdown
